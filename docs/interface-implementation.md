# Kubtel.ru: интерфейс этапа 5

Документ фиксирует frontend-срез первой версии после технического фундамента этапа 4. Цель этапа 5 - довести существующий UX-прототип и визуальную систему до связного адаптивного интерфейса без расширения бизнес-логики и без смены content-контрактов.

## Реализованный интерфейс

- Главная страница собирает первый коммерческий путь: hero, воронка, жизненные сценарии, услуги, тарифы, проверка адреса и FAQ.
- Страница тарифов показывает карточки, сравнительную таблицу и переход к заявке с выбранным тарифом.
- Страница подключения содержит структурную форму адреса, тарифа, опций и контакта.
- Страница поддержки разделяет аварийный сценарий, тарифные вопросы и новое подключение.
- Страница контактов показывает основные сценарии обращения до подтверждения финальных телефонов и офисных данных.
- Страница о компании остается смысловым каркасом до получения реальных фактов Kubtel.

## Компоненты этапа

- `ServiceCards` выводит карточки услуг из `services` collection, связывает их с тарифами и показывает статус достоверности данных.
- `Header` получил адаптивную навигацию: desktop menu и native mobile menu на `details/summary`.
- `StickyCta` остается нижней mobile-панелью и ведет в основную адресную воронку.
- `global.css` содержит UI-состояния карточек услуг, mobile-menu и легкие CSS-анимации появления.

## Контракты

- Схемы `src/content.config.ts` и доменные типы не менялись.
- Форма заявки не отправляет данные в CRM до этапа 6.
- Адресная проверка остается ручным прототипом до подключения адресной базы.
- Анимации реализованы через CSS и отключаются через `prefers-reduced-motion`.
- Сайт не зависит от тяжелого клиентского JS: интерактивность ограничена нативным меню и небольшим скриптом выбора тарифа в форме.

## Проверки закрытия

- `npm run format` выполнен.
- `npm run check` выполнен: 0 ошибок, 0 предупреждений, 0 hints.
- `npm run format:check` выполнен.
- `npm run build` выполнен.
- Bundled Playwright smoke выполнен на `/`, `/tariffs/`, `/connect/`, `/support/`, `/contacts/`, `/about/`.
- Viewport `390x844`, `768x1024`, `1440x1000`: горизонтальный overflow, console warnings/errors и мелкие критичные targets не обнаружены.
- Mobile/tablet navigation menu открывается и показывает основные ссылки.

## Ограничения

- Тарифы, зоны покрытия, SLA, телефоны, офисы и юридические тексты требуют подтверждения Kubtel.
- Реальные брендовые медиа и фотографии не получены.
- Интеграции заявки, CRM, Telegram/webhook и адресная нормализация относятся к этапу 6.
