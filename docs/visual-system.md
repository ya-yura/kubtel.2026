# Kubtel.ru: визуальная система этапа 3

Документ фиксирует первую версию визуальной системы поверх UX-прототипа этапа 2. Это рабочий дизайн-слой для разработки, а не финальный брендбук Kubtel.

## Визуальный DNA

Kubtel должен выглядеть как локальная технологичная команда: спокойно, понятно, без федерального глянца и без декоративного шума. Визуальный язык строится на светлой интерфейсной основе, графитовом технологичном контрасте и теплом оранжевом CTA.

Ключевые принципы:

- интерфейс сначала объясняет следующий шаг, потом продает;
- CTA заметны, но не спорят друг с другом;
- карточки держат плотность, не превращаясь в рекламные баннеры;
- статус данных виден как часть доверия, а не как техническая пометка;
- темные блоки используются как акцент для фактов и итогов, а не как основная тема.

## Палитра

| Токен | Цвет | Назначение |
| --- | --- | --- |
| `--color-ink` | `#15171a` | основной текст |
| `--color-graphite` | `#20242a` | темные технологичные блоки |
| `--color-muted` | `#5f6875` | вторичный текст |
| `--color-surface` | `#ffffff` | основная поверхность |
| `--color-soft` | `#f4f7fa` | спокойный фон секций и hover |
| `--color-accent` | `#f47b20` | главный CTA |
| `--color-success` | `#137a4b` | доступность и успех |
| `--color-info` | `#1f67b2` | информационные статусы |
| `--color-danger` | `#ba351f` | ошибки и недоступность |

Палитра намеренно не уходит в один цветовой тон: графит держит технологичность, оранжевый дает теплый локальный акцент, синий/зеленый/красный используются только для статусов.

## Типографика

- Базовый стек: `Inter`, системные sans-serif fallback.
- Размеры заданы в `rem` и меняются только через breakpoint, без зависимости от ширины viewport.
- Межбуквенный интервал в заголовках и служебных подписях равен `0`.
- Внутри карточек и compact UI заголовки ограничены меньшими размерами, чтобы не ломать плотные блоки.

## Сетка и поверхности

- Максимальная ширина страницы: `1180px`.
- Основной радиус: `8px`; мелкий радиус: `6px`.
- Секции остаются полноширинным потоком с ограниченным внутренним контейнером.
- Карточки используются для повторяемых элементов, форм, FAQ и контактных сценариев.
- Вложенные карточки не используются: внутренние состояния оформляются бейджами, строками выбора и легкими фонами.

## Компоненты

Визуальный слой покрывает компоненты первой версии:

- header с активным состоянием навигации;
- hero с адресным вводом и темным блоком фактов;
- тарифная карточка с featured-состоянием, ценой, доказательным статусом и CTA;
- таблица сравнения тарифов с горизонтальным скроллом и focus-состоянием;
- форма заявки с крупными полями, выбранным тарифом, опциями и юридической заметкой;
- state cards для проверки адреса;
- support pathway cards;
- FAQ accordion;
- mobile sticky CTA;
- contact cards на странице контактов.

## Состояния

Интерактивные элементы получают:

- `hover`: легкий подъем, усиление границы или фона;
- `active`: возврат элемента к плоскости;
- `focus-visible`: контрастное синее кольцо;
- `checked`: теплый фон и оранжевая граница;
- `invalid`: красная граница и мягкое error-кольцо;
- `aria-current="page"`: активная навигация.

## Адаптивность

Mobile-first слой начинается с одной колонки и крупных touch-targets от `44px`. На `720px` включаются desktop-навигация, многоколоночные сетки и скрывается нижний sticky CTA. На `980px` hero получает двухколоночную композицию.

## Микро-взаимодействия

Переходы ограничены `160-180ms`, используются только `transform`, `background-color`, `border-color` и `box-shadow`. Для `prefers-reduced-motion: reduce` анимации и плавный scroll отключаются.

## Контраст

Целевые пары подобраны под WCAG AA:

- основной текст `#15171a` на белом;
- вторичный текст `#5f6875` на белом;
- CTA-текст `#1a120b` на `#f47b20`;
- error `#ba351f` на `#fff0ec`;
- success `#137a4b` на `#e7f5ee`;
- info `#1f67b2` на `#edf5ff`.

Финальный брендовый аудит должен повторно проверить контраст после появления реальных фирменных материалов Kubtel.

## QA закрытия этапа

Результат технической визуальной проверки зафиксирован в `docs/visual-qa.md`.

Проверено:

- 6 ключевых страниц;
- mobile viewport `390x844`;
- desktop viewport `1440x1000`;
- HTTP-статусы;
- console warnings/errors;
- горизонтальный overflow;
- размеры критичных touch-targets;
- контраст основных цветовых пар.
